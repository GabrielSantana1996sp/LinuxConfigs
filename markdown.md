##  Regras de escrita no Markdown

### 1. Títulos
- Use `#` no início da linha.  
```markdown
# Título nível 1
## Título nível 2
### Título nível 3
```

### 2. Ênfase
- Negrito: `**texto**` → **texto**  
- Itálico: `*texto*` → *texto*  
- Riscado: `~~texto~~` → ~~texto~~  
- Código inline: `` `texto` `` → `texto`

### 3. Listas
- Não ordenada:
```markdown
- Item A
- Item B
  - Subitem
```
- Ordenada:
```markdown
1. Primeiro
2. Segundo
3. Terceiro
```

### 4. Links e imagens
```markdown
[Link para site](https://exemplo.com)
![Descrição da imagem](https://exemplo.com/imagem.png)
```

### 5. Citações
```markdown
> Isso é uma citação.
```

### 6. Linhas horizontais
```markdown
---
```

### 7. Blocos de código
```markdown
```
Texto sem destaque
```

```python
print("Olá, mundo!")
```
```

### 8. Tabelas
```markdown
| Nome | Idade |
|------|-------|
| Ana  | 25    |
| João | 30    |
```

### 9. Checklists (em GitHub, por exemplo)
```markdown
- [x] Tarefa concluída
- [ ] Tarefa pendente
```

---

##  Dicas de escrita
- O Markdown é **linear**: você escreve símbolos junto ao texto, sem menus ou botões.  
- É **minimalista**: poucos caracteres já mudam a formatação.  
- É **portátil**: funciona em várias plataformas (GitHub, VS Code, Obsidian, Reddit, etc.).  
- É **legível mesmo sem renderização**: se você abrir o arquivo puro, ainda entende o texto.
